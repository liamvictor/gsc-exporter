Cloud Shell Quickstart Guide

A step-by-step visual workflow and helper guide to run Google Search Console caching and reports inside Google Cloud Shell.

Introduction

Google Cloud Shell provides an command-line workspace that runs in your browser. Since Cloud Shell does not support opening browser windows automatically to handle Google OAuth, you must use our custom script helper to authorise GSC API access manually.

Follow the steps below to configure your Cloud Shell workspace and gather cache files automatically.

Step-by-Step Setup

Open Google Cloud Shell
  1. Prepare Environment Log into Cloud Shell and navigate to your project directory. Setup a Python virtual environment and install all dependencies: 
    cd ~/gsc-exporter
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
  2. Authorise GSC Access Because Cloud Shell runs on a remote server, execute our specialized authorisation script: 
    python utilities/auth-cloud-shell.py
    Follow the terminal steps:
    • Open the printed URL in your browser.
    • Select the Google Account having GSC permissions and grant access.
    • Copy the code parameter from the address bar of the redirected browser tab (the text after code=).
    • Paste this back into your Cloud Shell terminal.
  3. Define Sites List Create or edit the site-lists/sites.txt file in your workspace to list the GSC properties you wish to manage (one URL per line): 
    # Example format:
sc-domain:example.com
https://www.example.com/
  4. Execute Caching and Reporting Run the custom automation shell script to warm your GSC cache and check cache completeness automatically: 
    ./cloud-shell-automate.sh
    You can also run it silently with no user input using command arguments: 
    ./cloud-shell-automate.sh --non-interactive --run-reports
Warning: Remember that Cloud Shell will automatically shut down after 20–30 minutes of inactivity. Do not close your browser tab while the cache warmer is running.
