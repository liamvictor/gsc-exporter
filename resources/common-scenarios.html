Common Scenarios & Script Usage

This guide provides recipes for common analysis tasks, showing which scripts and commands to use, and how to interpret the results.

Goal: Compare this week's performance to last week's.

Script: performance-analysis.py

This script is designed specifically for comparing two time periods. You can add filtering flags (e.g., --page-contains or --query-contains) to narrow the analysis.

Command:
py performance-analysis.py https://www.example.com --last-7-days
Action:
  1. Run the command above for your site.
  2. Open the generated HTML report (e.g., performance-report-example-com-last-7-days-...html).
  3. Analyze the "Best Performing Content" and "Worst Performing Content" tables to see which pages have gained or lost the most clicks.
  4. Check the "Rising Stars" and "Falling Stars" for pages with dramatic changes in visibility.

Goal: Compare last month's performance to the same month last year.

Script: performance-analysis.py

The performance analysis script has a special flag for year-over-year comparisons. You can also add filters to narrow down the analysis.

Command:
py performance-analysis.py https://www.example.com --last-month --compare-to-previous-year
Action:
  1. The --compare-to-previous-year flag tells the script to use the same date range from the previous year as the comparison period.
  2. Open the report to see how your top pages have trended compared to twelve months ago, which is great for identifying seasonality and long-term growth.

Goal: Analyse performance for a specific site section, excluding brand queries.

Script: performance-analysis.py

Combine filtering flags to create a highly specific analysis. This is one of the most powerful features of the script.

Command:
py performance-analysis.py https://www.example.com --last-28-days --page-contains "/blog/" --query-not-contains "brand name"
Action:
  1. This command runs a performance analysis for the last 28 days.
  2. --page-contains "/blog/" limits the analysis to only your blog pages.
  3. --query-not-contains "brand name" removes all brand-related queries, focusing on pure organic discovery.
  4. The resulting report shows the top performing non-branded content in your blog section, which is invaluable for content strategy.

Goal: Find all keywords a specific blog post is ranking for.

Script: gsc-pages-queries.py

This script creates a detailed interactive report showing the relationship between pages and queries.

Command:
py gsc-pages-queries.py https://www.example.com --last-28-days
Action:
  1. Run the command. You can use any date flag, but --last-28-days gives a good recent overview.
  2. Open the generated HTML report.
  3. Click on the "Pages to Queries" tab.
  4. Use your browser's find function (Ctrl+F or Cmd+F) to locate your blog post's URL in the list.
  5. Click on the entry to expand it. The table inside will show every query that page received an impression for in the time period.

Goal: See all the pages that rank for my brand name.

Script: gsc-pages-queries.py

This is the reverse of the previous scenario, and you can use the same report to find the answer.

Command:
py gsc-pages-queries.py https://www.example.com --last-28-days
Action:
  1. Run the command and open the HTML report.
  2. Stay on the default "Queries to Pages" tab.
  3. Use your browser's find function (Ctrl+F or Cmd+F) to locate your brand name in the list of queries.
  4. Click on the query to expand it and see all the pages on your site that rank for it. This is useful for checking for brand keyword cannibalization.

Goal: See if a new article published yesterday is getting traffic.

Script: gsc-pages-queries.py

You can adjust the date range to be very recent to check on new content. Note that GSC data has a processing delay of about 48 hours, so `--last-24-hours` looks at data from two days ago.

Command:
py gsc-pages-queries.py https://www.example.com --last-24-hours
Action:
  1. Run the command.
  2. Open the report and switch to the "Pages to Queries" tab.
  3. Look for your new article's URL. If it's present, it has started getting impressions.
  4. If it's not there, don't worry. It can take a few days for new URLs to be crawled and start appearing in Search. Check again in a day or two.
