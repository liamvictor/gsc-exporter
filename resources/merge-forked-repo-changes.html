Git Workflow Guide

How to contribute changes from your fork back to the parent (upstream) repository.

Step 1: Push to your Fork

First, send your local commits to your own GitHub repository (the "origin").

git push origin main

Step 2: Create a Pull Request (PR)

The "Parent" repository (the original repo you forked from) is called the Upstream. To send your changes there:

  1. Go to your fork's page on GitHub: github.com/liamvictor/gsc-exporter.
  2. You should see a yellow bar saying "This branch is 1 commit ahead of liamdelahunty:main".
  3. Click Contribute > Open Pull Request.
  4. Review your changes and click Create Pull Request.

Step 3: Syncing after the PR is Merged

Once the owner of the parent repo accepts and merges your PR, your fork will technically be "behind" the parent because the parent now has a new merge commit. To fix this and stay perfectly in sync:

# 1. Fetch the latest changes from the parent repo
git fetch upstream

# 2. Merge the parent's changes into your local main branch
# This brings your local environment up to date with the parent
git merge upstream/main

# 3. Push the now-synced local main back to your fork
# This makes your GitHub fork match the parent exactly
git push origin main
Pro Tip: By following the "Sync" steps above, your fork will never show as being "behind" or "ahead" in a messy way. You'll always have a clean, straight line of history matching the original project.
