Date Flag Changes

Summary of Changes

The date flags across all three scripts (gsc_pages_exporter.py, performance-analysis.py, and snapshot-report.py) have been standardized to provide a consistent and comprehensive set of options. The following changes were made:

  • Added --start-date and --end-date flags to all scripts for custom date ranges.
  • Added a consistent set of flags for common time periods:
    • --last-24-hours
    • --last-7-days
    • --last-28-days
    • --last-month
    • --last-quarter
    • --last-3-months
    • --last-6-months
    • --last-12-months
    • --last-16-months
  • Removed the --compare- prefix from flags in performance-analysis.py.
  • Established clear default behaviors for each script when no date flag is specified.

New Flags

Flag Description Applies to
--start-date <YYYY-MM-DD> Specify a custom start date. All scripts
--end-date <YYYY-MM-DD> Specify a custom end date. All scripts
--last-24-hours Last 24 hours. All scripts
--last-7-days Last 7 days. All scripts
--last-28-days Last 28 days. All scripts
--last-month The last full calendar month. All scripts
--last-quarter The last full calendar quarter. All scripts
--last-3-months Last 3 months. All scripts
--last-6-months Last 6 months. All scripts
--last-12-months Last 12 months. All scripts
--last-16-months Last 16 months. All scripts

Default Behaviors

  • gsc_pages_exporter.py: Defaults to the previous calendar month.
  • performance-analysis.py: Defaults to comparing the last 28 days to the previous 28 days.
  • snapshot-report.py: Defaults to the last 3 months.
